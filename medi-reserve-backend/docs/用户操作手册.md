## 第一部分：患者端

### 1.1 注册账号

**功能说明**：新用户通过手机号 + 密码注册患者账号。

**操作步骤**：
1. 打开患者端页面，进入注册页
2. 填写注册信息：
   - **姓名**：真实姓名（非空）
   - **手机号**：用于登录和身份识别的唯一账号（非空，长度 11 位）
   - **密码**：登录密码（BCrypt 加密存储）
   - 可选信息：身份证号、性别
3. 点击「注册」按钮提交
4. 注册成功后自动返回患者 ID、姓名、手机号

**API 接口**：`POST /patient/register`

**注意事项**：
- 手机号全局唯一，已注册的手机号不可重复注册
- 密码采用 BCrypt 加密，平台无法反查明文密码

---

### 1.2 登录/退出

**功能说明**：使用注册时的手机号和密码登录系统。

**操作步骤**：
1. 打开患者端登录页
2. 输入手机号和密码
3. 点击「登录」
4. 登录成功后获得 JWT 令牌（有效期 2 小时），前端保存到本地
5. 后续所有需要登录的请求自动携带 Token
6. 退出时清除本地 Token

**API 接口**：`POST /patient/login`

**请求示例**：
```json
{
  "username": "13800000000",
  "password": "123456"
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "登录成功",
  "data": {
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "id": 1,
    "name": "张三",
    "phone": "13800000000"
  }
}
```

**注意事项**：
- 如果账号被禁用则无法登录
- Token 过期后需重新登录

### 1.3 浏览科室和医生列表

**功能说明**：无需登录即可查看科室列表和医生信息。

**操作步骤**：

**查看科室列表**：
- 进入首页或找医生页面，系统自动展示科室列表及每个科室下的医生数量
- 支持按科室筛选医生

**查看医生列表**：
- 支持按**科室**筛选
- 支持**关键词搜索**（医生姓名、擅长领域）
- 支持**分页浏览**

**查看热门医生**：
- 首页展示热度排名前 10 的医生（基于近 30 天评价 + 时间衰减算法）

**API 接口**：
| 接口 | 方法 | 路径 |
|------|------|------|
| 科室列表 | GET | `/patient/departments` |
| 医生列表 | GET | `/patient/doctors?department=&keyword=&page=1&size=10` |
| 热门医生 | GET | `/patient/doctors/hot` |
| 查看医生评价 | GET | `/patient/doctors/{doctorId}/evaluations` |

### 1.4 查看医生排班日历

**功能说明**：选择一个医生后，查看其未来 7 天的排班情况。

**操作步骤**：
1. 从医生列表中选择一个医生，进入医生详情页
2. 系统展示该医生未来 7 天的排班日历
3. 每天分为**上午**和**下午**两个时段
4. 每个时段显示剩余号源数量

**API 接口**：`GET /patient/doctors/{doctorId}/schedules`

**响应示例**：
```json
{
  "code": 200,
  "data": [
    {
      "scheduleDate": "2026-07-25",
      "morning": { "scheduleId": 101, "remainingCount": 8, "status": 1 },
      "afternoon": { "scheduleId": 102, "remainingCount": 3, "status": 1 }
    }
  ]
}
```

**注意事项**：
- 仅展示未来 7 天内有排班的日期
- 号源约满的时段标记为"已满"
- 停诊时段标记为"已停诊"

### 1.5 预约挂号

**功能说明**：患者选择排班时段，创建预约单。

**操作步骤**：
1. 在排班日历中选择一个可预约的时段，点击「预约」
2. 确认排班详情（医生信息、就诊日期、时段、剩余号源）
3. 点击「确认预约」
4. 系统创建预约单，返回预约单号和状态（待支付）

**API 接口**：
| 接口 | 方法 | 路径 |
|------|------|------|
| 查询排班详情 | GET | `/patient/schedules/{scheduleId}` |
| 创建预约 | POST | `/patient/appointments` |

**创建预约请求示例**：
```json
{
  "scheduleId": 101
}
```

**响应示例**：
```json
{
  "code": 200,
  "message": "预约创建成功，请在30分钟内完成支付，逾期将自动取消",
  "data": {
    "appointmentId": 1001,
    "appointmentNo": "APT20260723001",
    "status": 0,
    "statusText": "待支付",
    "payDeadline": "30分钟内"
  }
}
```

**技术特性**：
- 创建预约时使用**分布式锁（Redisson）**防止号源超卖
- 使用**乐观锁**扣减号源数量（`WHERE remaining_count > 0`）
- 如果号源不足，会提示"号源已被抢完，请重新选择"

**注意事项**：
- 每个患者同一排班只能预约一次
- 预约创建成功后有 **30 分钟支付等待期**，超时未支付自动取消

### 1.6 模拟支付

**功能说明**：模拟微信支付回调，将预约状态从"待支付"变更为"已支付"。

**操作步骤**：
1. 在「我的预约」列表中找到待支付的预约
2. 点击「去支付」
3. 系统模拟支付过程，直接变更状态为"已支付"

**API 接口**：`POST /patient/appointments/{appointmentId}/pay`

**注意事项**：
- 仅"待支付"状态的预约可支付
- 超时 30 分钟的预约会自动取消，无法支付
- 实际生产环境需对接微信支付 SDK（当前为模拟）

### 1.7 我的预约

**功能说明**：查看当前患者的所有预约记录，支持状态筛选。

**操作步骤**：
1. 进入「我的预约」页面
2. 可使用状态筛选器过滤：
   - 全部
   - 待支付（status=0）
   - 已支付（status=1）
   - 已就诊（status=2）
   - 已取消（status=3）
   - 已过期（status=4）
3. 每条预约显示：预约单号、医生姓名、科室、就诊日期、时段、状态
4. 待支付的可点击「去支付」
5. 已就诊的可点击「去评价」

**API 接口**：`GET /patient/appointments?status=0&page=1&size=10`

**注意事项**：
- 分页默认第 1 页，每页 10 条
- 需登录后才能查看

### 1.8 就诊评价

**功能说明**：患者对已就诊的预约进行评分和文字评价，支持匿名。

**操作步骤**：

**发表评价**：
1. 在「我的预约」中，找到已就诊的预约，点击「评价」
2. 填写评价内容：
   - **评分**：1-5 星评分（必填）
   - **评价内容**：文字评价，最多 500 字（选填）
   - **是否匿名**：可选择匿名展示（默认不匿名）
3. 点击「提交评价」

**查看我的评价**：
1. 进入「我的评价」页面
2. 查看所有已提交的评价
3. 可以删除评价（软删除，状态改为隐藏）

**API 接口**：
| 接口 | 方法 | 路径 |
|------|------|------|
| 创建评价 | POST | `/patient/evaluations` |
| 我的评价列表 | GET | `/patient/my-evaluations?page=1&size=10` |
| 删除评价 | DELETE | `/patient/evaluations/{evaluationId}` |

**注意事项**：
- 一个预约只能评价一次
- 只能评价状态为"已就诊"的预约
- 评价删除为软删除（隐藏），不会物理删除记录

### 1.9 在线问诊

**功能说明**：患者与医生进行实时聊天问诊。

**操作步骤**：

**进入问诊室**：
1. 在「我的预约」中找到已支付的预约，点击「进入问诊」
2. 系统获取问诊室信息（包含患者/医生信息、在线人数等）
3. 前端通过 STOMP over WebSocket 连接到 WebSocket 服务（端口 8084）
4. 订阅房间频道：`/topic/room/{appointmentId}`（按预约隔离）

**发送消息**：
- 输入消息内容，点击发送
- 消息通过 WebSocket 实时推送给对方
- 系统自动进行 XSS 过滤（`<` 和 `>` 转义）
- 消息按预约 ID 分房间隔离，隐私安全

**查看历史消息**：
- 支持分页加载历史聊天记录

**结束问诊**：
- 患者或医生均可主动点击「结束问诊」
- 结束后预约状态变为"已就诊"

**API 接口**：
| 接口 | 方法 | 路径 |
|------|------|------|
| 问诊室信息 | GET | `/consultation/room/{appointmentId}` |
| 聊天历史 | GET | `/consultation/history/{appointmentId}?page=1&size=20` |
| 结束问诊 | POST | `/consultation/end/{appointmentId}` |
| 发送消息 | WebSocket STOMP | `/app/chat.send` |

**WebSocket 连接信息**：
- 服务地址：`ws://localhost:8084/ws`
- 认证方式：通过 STOMP CONNECT 头携带 Token
- 消息推送：广播到房间频道 `/topic/room/{appointmentId}`

## 第二部分：医生端

### 2.1 注册与审核

**功能说明**：医生注册账号后需等待管理员审核，审核通过后方可登录。

**操作步骤**：

**注册**：
1. 打开医生端页面，进入注册页
2. 填写注册信息：姓名、手机号、密码、科室、职称、身份证号、性别、出生日期
3. 可选上传执业证书和资格证
4. 点击「注册」
5. 注册成功后进入**待审核**状态

**审核流程说明**：
| 审核状态 | 说明 | 能否登录 |
|---------|------|---------|
| 待审核（0） | 注册后默认状态，等待管理员审核 | 否 |
| 审核通过（1） | 管理员已通过审核 | 是 |
| 审核驳回（2） | 管理员已驳回，附驳回原因 | 否 |

**API 接口**：`POST /doctor/register`

**注意事项**：
- 审核状态为"待审核"或"驳回"时无法登录
- 审核状态可通过 `GET /doctor/profile/audit-status` 查询

---

### 2.2 登录

**功能说明**：审核通过的医生可以登录。

**操作步骤**：
1. 打开医生端登录页
2. 输入手机号和密码
3. 点击「登录」
4. 系统校验：账号是否存在 → 密码是否正确 → 审核状态是否为通过
5. 任一校验失败均返回对应错误提示

**API 接口**：`POST /doctor/login`

**注意事项**：
- 如果审核状态为"待审核"，提示"您的账号正在审核中，请耐心等待"
- 如果审核状态为"驳回"，提示驳回原因

### 2.3 排班管理

**功能说明**：医生可以设置自己的出诊排班。

#### 2.3.1 新增排班

**操作步骤**：
1. 进入「排班管理」页面
2. 选择日期和时段（上午/下午）
3. 输入最大挂号数（系统会基于历史数据**智能推荐**号源数量）
4. 点击「新增排班」

**API 接口**：`POST /doctor/schedules`

**请求示例**：
```json
{
  "scheduleDate": "2026-07-25",
  "period": 1,
  "maxCount": 20
}
```

**智能推荐号源说明**：
- 系统根据该医生同 weekday 的历史就诊数据，推荐当前排班的最大号源数
- 显示用户输入值与推荐值的差异百分比
- 医生可以接受推荐值，也可以自行修改

**API 接口（推荐）**：`GET /doctor/schedules/recommend?scheduleDate=2026-07-25&userInputMax=20`

#### 2.3.2 查询我的排班

**操作步骤**：
1. 进入「排班管理」页面
2. 默认展示所有排班记录，支持按日期范围筛选

**API 接口**：`GET /doctor/schedules?startDate=2026-07-25&endDate=2026-07-31`

#### 2.3.3 停诊/恢复排班

**操作步骤**：
1. 在排班列表中找到目标排班
2. 点击「停诊」→ 排班状态变为"已停诊"，该时段不再接受预约
3. 点击「恢复」→ 排班状态变为"正常"

**API 接口**：`PATCH /doctor/schedules/{id}/status?status=2`（停诊）/ `?status=1`（恢复）

#### 2.3.4 删除排班

**操作步骤**：
1. 在排班列表中找到目标排班
2. 点击「删除」
3. 仅当该排班下无预约记录时允许删除

**API 接口**：`DELETE /doctor/schedules/{id}`

**注意事项**：
- 已有预约记录的排班不可删除
- 停诊的排班已有预约的用户会收到通知（待开发）

**API 接口**：

| 聊天历史 | GET | `/consultation/history/{appointmentId}` |

### 2.5 个人信息编辑

**功能说明**：医生编辑个人资料和专业信息。

**操作步骤**：
1. 进入「个人中心」→「编辑资料」
2. 可修改字段：
   - **普通信息**：姓名、手机号、性别、身份证号（修改后立即生效）
   - **擅长领域**、**个人简介**（修改后立即生效）
   - **证件信息**：执业证书、资格证（上传后需管理员审核，审核通过后生效）
3. 点击「保存」

**API 接口**：
| 接口 | 方法 | 路径 |
|------|------|------|
| 修改个人信息 | PUT | `/doctor/profile` |
| 查询证件审核状态 | GET | `/doctor/profile/audit-status` |

**证件上传**：
- 通过 `GET /doctor/oss/sts-token` 获取阿里云 OSS STS 临时凭证
- 使用临时凭证直接将文件上传到阿里云 OSS
- 上传后记录 URL，提交管理员审核

**注意事项**：
- 证件审核状态可实时查询
- 审核驳回时附驳回原因，医生可重新提交

### 2.6 数据统计

**功能说明**：医生查看个人运营数据统计。

#### 2.6.1 总览统计

显示：
- 总接诊数
- 好评率
- 平均评分
- 今日接诊数
- 待处理问诊数

**API 接口**：`GET /doctor/statistics/overview`

#### 2.6.2 每日接诊趋势

- 近 N 天每日接诊量柱状图，默认 7 天，最大 90 天

**API 接口**：`GET /doctor/statistics/trend?days=7`

#### 2.6.3 评价列表

- 分页查看患者对医生的所有评价

**API 接口**：`GET /doctor/statistics/evaluations?page=1&size=10`

## 第三部分：管理端

### 3.1 登录

**功能说明**：管理员通过隐藏入口登录管理后台。

**操作步骤**：
1. 访问管理端隐藏入口：`/login?role=admin`
2. 输入用户名和密码
3. 系统校验后返回 JWT 令牌和角色信息

**初始管理员账号**：

| 用户名 | 密码 | 角色 |
|--------|------|------|
| `admin` | `123456` | 超级管理员（拥有所有权限） |
| `admin_normal` | `123456` | 普通管理员（仅有查看权限） |

**API 接口**：`POST /admin/login`

**注意事项**：
- 账号被禁用则无法登录
- 超级管理员和普通管理员看到的菜单和功能不同

### 3.2 医生审核

#### 3.2.1 查看待审核列表

**操作步骤**：
1. 登录管理端，进入「审核管理」→「医生审核」
2. 查看所有待审核的医生列表（分页，按注册时间升序）
3. 列表显示：医生姓名、手机号、科室、职称、注册时间

**API 接口**：`GET /admin/doctors/pending?page=1&size=10`

#### 3.2.2 查看审核详情

**操作步骤**：
1. 在待审核列表中选择一位医生
2. 查看完整的注册信息和审核资料（擅长领域、个人简介、证件图片）
3. 根据资料真实性做出审核决定

**API 接口**：`GET /admin/doctors/{id}/audit-detail`

#### 3.2.3 审核通过

**操作步骤**：
1. 确认医生资料无误后，点击「审核通过」
2. 系统更新审核状态为"通过"，医生即可登录

**API 接口**：`PATCH /admin/doctors/{id}/approve`

#### 3.2.4 审核驳回

**操作步骤**：
1. 确认医生资料有问题后，点击「审核驳回」
2. 在弹出的输入框中填写驳回原因（必填）
3. 系统更新审核状态为"驳回"，医生登录时能看到驳回原因

**API 接口**：`PATCH /admin/doctors/{id}/reject`

**请求示例**：
```json
{
  "rejectReason": "执业证书照片模糊，请重新上传清晰版"
}
```

#### 3.2.5 证件变更审核

**操作步骤**：
1. 进入「证件审核」页面，查看已变更证件的医生列表
2. 查看证件详情
3. 审核通过：新证件立即生效，替换旧证件
4. 审核驳回：清空待审核数据，医生可重新提交

**API 接口**：
| 接口 | 方法 | 路径 |
|------|------|------|
| 待审核证件列表 | GET | `/admin/cert-pending` |
| 证件详情 | GET | `/admin/{doctorId}/cert-pending-detail` |
| 审核证件 | PATCH | `/admin/{doctorId}/cert-audit` |

## 附录：错误码一览

| 错误类型 | 说明 |
|---------|------|
| 账号不存在 | 手机号/用户名未注册 |
| 密码错误 | 输入的密码不匹配 |
| 账号已禁用 | 管理员禁用了该账号 |
| 账号审核中 | 医生注册后等待审核 |
| 审核已驳回 | 医生审核被驳回 |
| 号源不足 | 排班剩余号源为 0 |
| 重复预约 | 同一患者同一排班已预约 |
| 预约超时 | 30 分钟未支付已自动取消 |
| 已支付 | 预约已支付，不可重复支付 |
| 不可评价 | 预约状态非"已就诊"不可评价 |
| 重复评价 | 一个预约只能评价一次 |
| Token 过期 | JWT 令牌已过期，需重新登录 |
| 权限不足 | 当前角色无此操作权限 |
